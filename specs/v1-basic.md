# Claude Code Prompt — v1-basic

Run this from the repo root. Read `specs/Design.md` first for the full rationale; this prompt is the build
spec for the **v1-basic** folder only.

---

Build the `v1-basic` version of the blog demo: pure HTML, CSS, and JavaScript with **no build step
and no dependencies**. Someone must be able to open `v1-basic/index.html` directly in a browser and
have everything work.

## Source content

The canonical posts are the 8 markdown files in `/content` (frontmatter: `title`, `date`, `tags`,
`excerpt`, `slug`). Read them and use their real titles, dates, tags, excerpts, and body text. Do not
invent new posts. For v1 you are converting this markdown into static HTML at author-time (i.e. you
generate the HTML now; the page does not parse markdown at runtime).

## Requirements (verifiable)

1. `v1-basic/index.html`
   - A header containing the site title and a **global like total** element.
   - A controls row: a text `input` for **search** and a `select` for **sort** (options: Newest,
     Oldest, Title A–Z).
   - A grid of **8 post cards rendered as static HTML in the markup** (not generated by JS at
     runtime). Each card shows title, formatted date, tag(s), excerpt, a **like button with a count**,
     and a "Read" link to that post's page.
2. `v1-basic/styles.css`
   - Styles the header, controls, card grid, and cards. Clean and legible; responsive enough to not
     break on a projector. No frameworks.
3. `v1-basic/script.js` — progressive enhancement over the existing DOM:
   - **Search**: filters visible cards by title as the user types.
   - **Sort**: reorders the card nodes in the DOM by the selected option.
   - **Likes**: clicking a card's like button increments that card's count, updates the global total
     in the header, and persists both to `localStorage`. On load, restore counts from `localStorage`.
   - All of this manipulates the existing DOM nodes by hand — do not rebuild the list from a JS data
     array. The point is to show manual DOM wiring.
4. `v1-basic/posts/<slug>.html` — one **separate static HTML page per post** (8 total), each with the
   post's full body, a link back to the index, and the same CSS. Navigation is a full page reload.

## Constraints

- Zero dependencies, zero build. No `package.json` in this folder.
- Content must be present in the HTML source (view-source shows the post text) — this is a
  deliberate contrast with v2.
- Keep it readable: this code will be shown on screen and explained live.

## Acceptance check

- Opening `v1-basic/index.html` via `file://` shows 8 cards with no console errors.
- Search narrows the list; sort reorders it; liking a card updates both its count and the header
  total and survives a refresh.
- Clicking "Read" loads a separate post page that shows the full body.
